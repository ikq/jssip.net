--- title: "2014-11-03: JsSIP 0.5.0, let's Node!" --- h1. 2014-11-03: JsSIP 0.5.0, let's Node! The new version 0.5.0 of JsSIP runs in Node.js! This means that JsSIP can be used within your Node application as a SIP WebSocket client: CODE_BEGIN "bash" $ npm install jssip CODE_END CODE_BEGIN "javascript" var JsSIP = require('jssip'); var ua = new JsSIP.UA(...); ua.on('registered', function() {...}); [...] ua.start(); CODE_END Take a look to the new configuration parameter <%= my_lib_link_to "/documentation/api/ua_configuration_parameters/#parameter_node_ws_options", {:text=>'node_ws_options'} %>. h2. Browserified! The internal design of JsSIP has also been modified, becoming a real Node project in which the "browser version" (@jssip-0.5.0.js@ or @jssip-0.5.0.min.js@) is generated with "@browserify@":http://browserify.org. This also means that the browser version can be loaded with AMD or CommonJS loaders.
